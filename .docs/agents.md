## 5. Технические особенности расширения
*   **Метод отслеживания глав:** Реализован через лайки Boosty (`isLiked`). При фоновом обновлении лайкнутые посты автоматически помечаются как просмотренные.
*   **Авторизация API:** Запросы к `api.boosty.to` отправляются с заголовком `Authorization: Bearer <token>`, где токен считывается из `localStorage` (ключ `auth`, поле `accessToken`). Это необходимо для корректного получения статуса `isLiked`.
*   **Автономность разработки:** Доступ к закрытому платному контенту не требуется для разработки, так как метаданные заголовков глав и теги авторов получаются через публичное API Boosty.
*   **SPA-маршрутизация и видимость:** Сайт Boosty функционирует как SPA. Расширение загружается на всех страницах `https://boosty.to/*`, динамически скрывая/показывая интерфейс при изменении URL (через перехват истории переходов и резервный таймер) в зависимости от того, находится ли пользователь на странице автора `lightfoxmanga`.
*   **Автозакрытие панели:** Реализовано закрытие боковой панели при клике в любую область страницы за пределами самой панели и кнопки её вызова.
*   **Защита от «Extension context invalidated»:** Все обработчики событий и вызовы Chrome API обёрнуты проверкой `isExtensionContextValid()`. При инвалидации контекста (обновление/перезагрузка расширения) вызывается функция `cleanup()`, которая останавливает интервалы, отписывается от всех слушателей событий, восстанавливает оригинальные `history.pushState`/`replaceState` и удаляет DOM-элементы.
*   **Масштаб по умолчанию:** 125% (`state.settings.zoom`).
*   **Стилизация чекбоксов:** Галочки прочитанных глав окрашены в ярко-зелёный (`accent-color: var(--lf-color-green)`). Пролайканные главы получают класс `.lf-liked-checkbox` с `pointer-events: none` вместо атрибута `disabled` для сохранения цвета.
*   **Сворачивание категорий:** По умолчанию все группы тайтлов свёрнуты. Логика: `state.collapsedGroups[groupName] !== false` означает «свёрнуто».
