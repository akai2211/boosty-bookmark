# Архитектура и UI (Изначальный план реализации)

Этот проект представляет собой расширение для Google Chrome (Manifest V3), разработанное специально для страницы автора `lightfoxmanga` на Boosty. Расширение помогает зрителям отслеживать прогресс просмотра видеороликов и озвучек, организовывать тайтлы по категориям и оставлять личные заметки (блокнот).

Интерфейс встраивается в правую часть экрана в виде аккуратной выдвижной панели, оформленной в едином стиле с Boosty (поддержка темной и светлой темы, лаконичный дизайн без лишней перегруженности).

---

## Ответы на вопросы и архитектурные решения

### 1. Как отслеживать просмотренное (Интеграция с Boosty)
*   **Через лайки (`isLiked`):** 
    В API Boosty каждый пост содержит флаг `isLiked: true/false` для текущего авторизованного пользователя. Так как этот флаг возвращается прямо в списке постов (по 100 штук за раз), мы можем **мгновенно и без лишних запросов** узнать, какие посты пользователь лайкнул.
    Мы добавим опцию: *«Считать лайкнутые посты просмотренными»*. Если она включена, все посты, которым пользователь поставил лайк на Boosty, автоматически помечаются как просмотренные. Это обеспечит идеальную совместимость с вашей историей на любых устройствах!
*   **Почему не комментарии:**
    Для проверки комментариев расширению пришлось бы делать отдельный сетевой запрос для каждого из 1250+ постов. Это заняло бы несколько минут и привело бы к временной блокировке вашего IP-адреса со стороны Boosty за превышение лимитов запросов (Rate Limit). Лайки работают мгновенно.
*   **Локальные чекбоксы:**
    Пользователь всегда может вручную ставить/снимать галочки прямо в интерфейсе расширения. Это состояние сохраняется локально.

### 2. Проблема подписки при разработке (Аккаунт не требуется!)
*   **Передавать аккаунт НЕ нужно.** 
    Вся метаинформация о постах (заголовки вида *«Божественный убийца я тень 1-10»*, теги, даты публикации и требуемые уровни подписки) отдается API Boosty **публично** (даже без авторизации). Мы это проверили тестами.
    Скрыто только само содержимое (плеер с аудио), но для работы трекера и блокнота плеер не нужен — нам достаточно заголовков постов и тегов. Когда расширение будет работать у вас (где вы авторизованы и имеете подписку), браузер автоматически подставит ваши авторизационные данные для лайков.

### 3. Автоматическая группировка по тирам подписки
Каждый тайтл (группа постов по уникальному тегу) автоматически распределяется по выпадающим спискам на основе метаданных постов:
*   **Только для девушек:** Если в названии постов содержится фраза *«ТОЛЬКО ДЛЯ ДЕВУШЕК»* или тег содержит слово *«девушек»*.
*   **Любителям пика:** Если тег равен *«пик боевых искусств»* или уровень доступа большинства постов — *«Любители пика💥»*.
*   **Любителям ютуба:** Если уровень доступа постов — *«Любитель ютуба»*.
*   **Любителям манги:** Если уровень доступа постов — *«Любитель манги😈»*.
*   **Лисямбы мои:** Если уровень доступа постов — *«Лисямбы🦊»*.
*   **Для шейхов:** Если уровень доступа постов — *«Массонский орден шейхов💎»*.
*   **Бесплатные / Без тега:** Попадают в категорию «Бесплатные».

## Структура интерфейса панели (Справа)

Интерфейс состоит из следующих разделов (сверху вниз):

1.  **Поиск и Синхронизация:** Поле для быстрого поиска тайтлов по названию и кнопка принудительного обновления БД.
2.  **Секция «Новые тайтлы»:** Тайтлы, которые появились в базе с момента вашего последнего визита.
3.  **Секция «Новые главы»:** Тайтлы из списков «Избранное»/«Смотрю», у которых вышли новые посты с момента вашего последнего захода.
4.  **Разделы статусов:**
    *   `Избранное` (разворачиваемый список)
    *   `Смотрю` (разворачиваемый список)
5.  **Разделы по уровням подписки (Категории с выпадающими списками):**
    *   `Все` (общий плоский список всех тайтлов)
    *   `Любителям ютуба`
    *   `Любителям манги`
    *   `Только для девушек`
    *   `Любителям пика`
    *   `Для шейхов`
    *   `Лисямбы мои`
    *   `Бесплатные`
6.  **Секция «Брошено»:** Тайтлы, добавленные сюда, скрываются из всех остальных списков, чтобы не мешать.

### Цветовая индикация тайтлов (Круглые маркеры):
*   🟡 **Желтый:** Есть новые или еще непросмотренные посты/главы.
*   🟢 **Зеленый:** Все вышедшие главы полностью просмотрены (отмечены галочками или лайками).
*   ⚪ **Серый:** Просмотр тайтла еще не начат (ни один пост не отмечен).
*   🔴 **Красный:** Тайтл находится в категории «Брошено».

## Детальный вид тайтла (При клике на название)

Когда пользователь кликает на тайтл, панель переключается на детальный вид:
1.  **Кнопка «Назад»** к общему списку.
2.  **Название тайтла** (со ссылкой, ведущей на отфильтрованную страницу тега на Boosty).
3.  **Панель управления статусом:** Выпадающий список со смайликами для быстрого переноса в: *Избранное (⭐) / Смотрю (🟡) / Завершено (🟢) / Брошено (🔴) / Не отслеживаю (⚪)*.
4.  **Блокнот (Заметки):** Текстовое поле индивидуально для этого тайтла. Все изменения сохраняются автоматически по мере ввода (через `debounce` в локальное хранилище).
5.  **Список постов (Главы):**
    *   Каждый пункт — это конкретный пост на Boosty (например, *«Божественный убийца я тень 11-20»*).
    *   Слева от каждого пункта — чекбокс (если отмечен, значит просмотрено).
    *   Название поста кликабельно — при клике открывается страница поста в новой вкладке (мы можем автоматически ставить галочку при переходе).
    *   Сортировка: кнопка переключения порядка глав (Сначала старые / Сначала новые).
