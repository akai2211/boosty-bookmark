## Идеи и планы на будущее

1. **Синхронизация с онлайн-сервисом (Поэтапная реализация):**
   Добавить возможность синхронизации прогресса (настроек, отслеживаемых тайтлов, таймстаппов) через облако.
   - [x] **Этап 1: Синхронизация через WebDAV (Nextcloud / универсальный WebDAV)** — ZIP-слияние, UI (адрес, имя, код доступа), автосинхронизация при открытии панели. Без Яндекс.Диска (для него — OAuth в будущем).
   - [ ] **Этап 2: Нативная синхронизация Google Drive (для Chrome)** — интеграция с `chrome.identity`, OAuth2 авторизация в один клик.
   - [ ] **Этап 3: Портирование Google Drive под Firefox** — переход на `browser.identity.launchWebAuthFlow` для авторизации в Firefox.

2. **Мультиавторская (универсальная) поддержка и концепция шаблонов (Templates):**
   Сделать расширение универсальным, добавив поддержку не только текущего автора (LightFox), но и других авторов на Boosty:
   *   Реализовать **систему шаблонов парсинга (Parsing Templates)** под разных авторов. Так как авторы выкладывают контент по-разному, шаблоны позволят адаптировать логику извлечения номеров глав, парсинга тайтлов и распределения по подпискам под конкретный блог.
   *   Шаблоны будут динамически переключаться на основе `slug` блога (например, свой шаблон для `lightfoxmanga` и свои правила для других авторов), сохраняя единую архитектуру ядра и интерфейса сайдбара.
   *   Обеспечить гибкое развертывание библиотеки на любых блогах Boosty, где применима концепция структурированного отслеживания прогресса.

3. **Подготовка к модерации в интернет-магазинах:**
   Проверить и подготовить документацию расширения (описания, политику конфиденциальности и др.) к проверке при публикации в Chrome Web Store и других каталогах.

4. **Тестирование расширения в браузере Firefox:**
   Провести ручное тестирование и отладку функционала расширения в браузере Firefox (синхронизация, рендеринг, копирование адресов донатов) перед публикацией релиза.
