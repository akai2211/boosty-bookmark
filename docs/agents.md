# Память агента: Инструкция и правила (Boosty Bookmark)

Данный файл является главным сводом правил, предпочтений пользователя и ограничений для ИИ-агентов, обязательным для соблюдения во всех сессиях разработки.

---

## 1. Языковые настройки и стиль общения
*   **Имя пользователя:** Акай (Akai). Всегда обращаться по этому имени и использовать **мужской род** (мужские грамматические формы).
*   **Язык общения:** Строго русский язык (в ответах пользователю, комментариях к коду и коммитах).
*   **Стиль принятия решений:** Никогда не спешить. Всегда запрашивать явное согласие пользователя перед важными изменениями (переименования, удаление, рефакторинг). Предлагать альтернативные варианты.
*   **Тон ответов:** Лаконичный, технически точный, без лишней вежливости и самовосхваления.

---

## 2. Верификация памяти и актуализация проекта
*   **Правило верификации:** Перед началом работы над любой задачей (в новой сессии или чате) ИИ-ассистент **обязан** сверить содержимое файлов памяти (`docs/memory.md` и глобального KI) с реальным кодом проекта. При обнаружении расхождений (например, функция числится в планах, но уже написана) необходимо сначала предложить пользователю актуализировать память, и только потом приступать к реализации новых функций.
*   **Правило обновления памяти и коммита:** При получении команды от Акая *«обнови память и закоммит»* (или аналогичной), ИИ-ассистент обязан автоматически актуализировать все файлы документации в папке `docs/` (внести выполненные задачи в `tasks.md`, обновить описание изменений в `technical_notes.md` и добавить запись о новом коммите в раздел `5. Выполненные коммиты` в `memory.md`), запустить тесты для проверки работоспособности и только после этого сделать локальный коммит.

---

## 3. Правила работы с Git и сборкой (КРИТИЧЕСКИ ВАЖНО)
*   **Запрет на автоматический push:** **НИКОГДА** не выполнять команду `git push` или любые другие команды, отправляющие изменения в удаленный репозиторий, автоматически. Это действие выполняется **только** после прямого указания Акая (*«сделай пуш»*, *«запушь»*).
*   **Запрет на автоматический коммит:** **НИКОГДА** не делать локальные коммиты (`git commit`) без прямого указания Акая (*«закоммить»*, *«сделай коммит»*).
*   **Правило запуска тестов:** Запуск тестов (`npm test`, `playwright`, `vitest` и др.) разрешен **только перед коммитом** (после получения команды закоммитить) для валидации изменений перед коммитом. В остальных случаях запускать тесты без прямой необходимости запрещено.
*   **Временный запрет на повышение версии:** Пока расширение находится на этапе разработки и тестирования (до официального релиза), ИИ-агентам **запрещено самостоятельно повышать версию** расширения. Версия должна оставаться фиксированной (`0.9.0`). Повышение версии допускается только после прямой команды Акая (*«подними версию»*, *«готовим релиз»*).
*   **Правило обновления версии при релизе:** При подготовке релиза (по указанию Акая) ИИ-агент обязан убедиться в актуальности версии в `manifest.json` и запустить скрипт сборки (`npm run build`), чтобы автоматически синхронизировать версию в `manifest.firefox.json` and `package.json`. Устанавливать мажорные версии (например, `1.0.0`) имеет право **только Акай**.
*   **Информационные действия разрешены:** Выполнение вспомогательных команд `git add` (для индексации), `git status`, `git diff`, `git log` разрешено без предварительного согласования.

## 4. Названия и структура папок проекта
*   **Имя расширения:** **Boosty Bookmark** (заголовок выдвижной панели в `content.js` и в `manifest.json`).
*   **Имя папки проекта на диске:** `/home/akai/projects/boosty-bookmark`
*   **Служебные файлы и отчеты:** Все служебные файлы (планы, отчеты, задачи) хранятся в папке `docs/` внутри проекта и отслеживаются Git.
*   **Локализация и интерфейс:** Файл `docs/ru_interface_structure.md` является вспомогательным рабочим файлом пользователя (Акая) для разработки структуры интерфейса. ИИ-агенты **не должны** изменять этот файл и **не должны** переносить из него новые ключи/значения локализации в `locales.js`, пока пользователь прямо не попросит об этом.
*   **Папка архива:** Папка `docs/archive/` содержит старые исторические материалы (исходное ТЗ). ИИ-агентам **запрещено** читать файлы из этой папки, чтобы не забивать контекстное окно и токены, кроме случаев, когда пользователь напрямую попросит об этом.
*   **Папка описания фич:** Папка `docs/features/` содержит подробную документацию по конкретным реализованным функциям. ИИ-агенты **должны игнорировать** файлы из этой папки при первичном входе в контекст проекта и не загружать их в память без необходимости. Обращаться к ним следует **только** в том случае, если соответствующая функция сломалась и требует исправления/доработки.
