## История изменений (вехи)

Ниже — укрупнённые вехи разработки. **Полная история коммитов — в `git log`** (дублировать её здесь не нужно).
Технические подробности каждой вехи см. в `technical_notes.md` (раздел указан в скобках).

* **Force Video Quality для VK-плеера (починен, механизм переписан)** — старый подход (клики по меню плеера) нереализуем: кнопку настроек нельзя открыть без trusted-клика. Новый: `page_script.js` перехватывает `localStorage.getItem('vk_player_preferred_quality')` и подставляет актуальный `videoId` плеера (`vk-video-player.store.videoId`) + качество; плеер сам стартует в нужном разрешении. Настройка передаётся из content через `postMessage` (`LF_SET_QUALITY_PREF`), разморожена в UI, проверена вживую. (tasks.md §6; technical_notes §39)
* **Группа «Просмотрены все главы» и динамический top сайдбара** — добавлена группа «Просмотрены все главы» на вкладке «Смотрю» для тайтлов с 100% прогрессом; реализовано динамическое позиционирование сайдбара и дев-панели через `ResizeObserver` на хедер сайта (`#TopMenu`). (tasks.md §27; technical_notes §38)
* **Надёжный автоскролл к главе в ленте** — `checkAndScrollToPost` переписан в двухфазный цикл (догрузка постов кликом «Загрузить еще» + скролл к концу колонки постов с «толчком» при парковке → приземление со стабилизацией). Чинит провал на тайтлах >10 постов. Опция `openChaptersInFeed` теперь вкл по умолчанию, бета снята. (tasks.md §26; technical_notes §21)
* **Надёжная полная синхронизация** — `fetchPostsPage()` с ретраями/backoff (переживает rate-limit Boosty) + докачка с чекпоинта (`lf_full_sync_resume_*`) при обрыве. Исправляет баг, когда единичный сбой `fetch` обнулял всю базу. (tasks.md §25; technical_notes §37)
* **Модульный рефакторинг `content.js` → `src/`** — монолит (5600+ строк) разбит на 12 ES-модулей, сборка через esbuild, dev-код вырезается флагом `DEV` + tree-shaking. Завершён и слит в `main`. (technical_notes §30; исторический план — `archive/split_plan.md`)
* **Автопереход к тайтлу по тегу Boosty** — `syncActiveTitleFromUrl()` открывает детальный вид при выборе тега на сайте. (§29)
* **Односторонняя синхронизация лайков: лайк только на посте → галочка (переписана 2026-06)** — лайкать можно ТОЛЬКО на самом посте Boosty; расширение лайки не пишет, лишь читает. `page_script.js` (MAIN-world) перехватывает fetch/XHR `/reaction` и шлёт `LF_REACTION_INTERCEPTED` → `handleInterceptedReaction` обновляет `post.isLiked` и галочку в реальном времени. При `syncLikes` ВКЛ все чекбоксы read-only (класс `lf-checkbox-locked` + блокировка `click`→`preventDefault`, цвет обычный — не серый `disabled`; отметка только лайком); при ВЫКЛ — лайки и галочки независимы. Удалены `sendBoostyReaction`/`removeBoostyReaction`, `LF_SET_LIKE_VISUAL`, `LF_SYNC_ALL_LIKES`, вся косметика сердечка в page_script. Детали — `features/like_checkbox_sync.md`. (§13–15)
* **Внутренний build-маркер + кнопка «Обновить расширение»** — `LF_INTERNAL_BUILD` (формат `ГГГГ-ММ-ДД.N`) в `src/utils.js` и `page_script.js`; виден в дев-панели, баннерах загрузки и как `window.__LF_BUILD`. В дев-панели — кнопка «⟳ Обновить расширение» (`RELOAD_EXTENSION` → `background.js` → `chrome.runtime.reload()` + перезагрузка вкладки).
* **Локализация RU/EN** — словарь `locales.js`, выбор языка через куку `locale` (`getCurrentLang`), `tCategory` для тиров. (§36)
* **Облачная синхронизация WebDAV** — ZIP-формат, слияние по `updatedAt`, обход CORS/CSP через `background.js`, провайдеры Яндекс/универсальный WebDAV. (§24, §35)
* **Панель DevTools + скрипт сборки** — эмуляция даты отсечки, скрытие блока «Об авторе»; релизная сборка `build.cjs` (Chrome/Firefox). (§30.3–30.4, §27)
* **Портирование под Firefox** — `manifest.firefox.json`, `npm run build:firefox`. (§27)
* **Раздел «О расширении» и поддержка** — кнопки Boosty/USDT (TRC-20), QR, защита реквизитов от подмены в DOM через `MutationObserver`. (§26)
* **Медиаплееры** — автозапоминание времени audio/video, принудительное качество видео VK (сейчас заморожено из-за смены плеера Boosty). (§25)
* **Сброс данных** — двухэтапное подтверждение, полная очистка `chrome.storage.local`. (§20)
* **Бизнес-логика библиотеки** — авто-статусы, группы завершённости, вкладка «Новые»/`lastVisit`, категории «Все»/«Бесплатные»/«Объявления», «красивые имена», сортировка. (§7, §23, §28, §31–§33)

## Автоматизированное тестирование

### Инфраструктура
* **Среда:** Vitest (юнит-тесты, окружение `jsdom`) + Playwright (E2E).
* **Конфигурация:** `vitest.config.js` — `include: ['**/*.test.js']` (Vitest берёт только `*.test.js`, игнорируя E2E `*.spec.js`) и `define: { DEV: 'false' }`. Папка `test-results/` в `.gitignore`.
* **Скрипты (`package.json`):** `test:unit` (Vitest), `test:e2e` (Playwright), `test` (оба последовательно). Хук `pretest` собирает свежий бандл (`build:js`) — E2E грузит `content.js` из корня.

### Юнит-тесты
* `tests/unit.test.js` — ESM; `src/content.js` грузится динамическим `await import()` после установки моков `global.chrome`/`sessionStorage`. Покрывают `formatDate`, `arePostsEqual`, `ensureUserData`, группировку `getGroupedTitles` (фильтрация техтегов, «Объявления»).
* `tests/webdav-sync.test.js` — слияние локального и облачного состояний: заметки/статус по `updatedAt`, прочитанные главы — tombstone-логика (`mergeReadState`: `readMarks`/`unreadMarks`, синхронизация снятия отметки), настройки по `settings.updatedAt`, новинки LWW по `newListsUpdatedAt`, max по `playerTimestamps`. Детали слияния — `technical_notes.md` §41.

### E2E-тесты (`tests/extension.spec.js`)
Запуск во временном контексте Chrome с загруженным расширением; сеть (HTML-страница Boosty и `api.boosty.to`) мокируется для автономности. **8 сценариев:** инициализация триггера/сайдбара; синхронизация и отображение (вкладка «Все», чекбоксы по `isLiked`); поиск и фильтрация; блокнот (автосохранение по `blur`); смена статусов; очистка новинок с эмуляцией даты в DevTools; сброс данных с двухэтапным подтверждением; модальное окно USDT и QR. Детали сценариев — в самом файле теста.
