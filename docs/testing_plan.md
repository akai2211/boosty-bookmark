# План тестирования расширения (Testing Plan) — Boosty Bookmark

В данном документе представлен план внедрения автоматизированного тестирования для Chrome-расширения **Boosty Bookmark**. Тестирование позволит гарантировать стабильность работы UI, корректность парсинга постов и надежность двусторонней синхронизации.

---

## 1. Стратегия тестирования

Для полной проверки расширения предлагается использовать двухуровневую структуру тестов:

### Уровень 1: Юнит-тесты (Unit Tests)
*   **Цель:** Проверка чистых логических функций без запуска браузера (быстрое выполнение).
*   **Что тестируем:**
    *   Функции группировки постов по тайтлам (выделение названий, очистка от спецтегов из `TAGS_BLACKLIST`).
    *   Определение тира подписки для каждого тайтла на основе метаданных.
    *   Парсинг описания блога автора («Об авторе») и связывание ID тегов с красивыми названиями.
    *   Логику сортировки тайтлов (по алфавиту, дате, прогрессу и количеству глав).
    *   Логику авто-статуса «Смотрю» при превышении лимита прочитанных глав.
*   **Стек:** **Vitest** (или Jest) + **JSDOM** (для базовой эмуляции DOM).

### Уровень 2: E2E-тесты (End-to-End Tests)
*   **Цель:** Проверка реального поведения расширения в браузере Chrome, включая внедрение скриптов и SPA-навигацию.
*   **Что тестируем:**
    *   Появление плавающей кнопки-триггера при переходе на страницу автора и её скрытие на посторонних страницах.
    *   Открытие/закрытие сайдбара и отображение индикаторов загрузки/ошибок.
    *   Интерактивное взаимодействие: выбор вкладок, раскрытие папок, переключение сортировок.
    *   Работа настроек: изменение масштаба (zoom), перетаскивание порядка вкладок (Drag & Drop), экспорт/импорт настроек, кнопка сброса.
    *   **Синхронизация лайков:** Проверка перехвата кликов по кнопке лайка на сайте и обновление чекбокса в сайдбаре (интеграция `content.js` и `page_script.js`).
    *   **SPA-навигация и автоскролл:** Бесшовный переход по ссылкам тайтлов/глав и скролл к ленте или конкретному посту.
*   **Стек:** **Playwright** (Chromium). Playwright имеет встроенную поддержку тестирования расширений Chrome посредством запуска браузера с флагом `--disable-extensions-except`.

---

## 2. Архитектура тестового окружения

Поскольку расширение полностью зависит от платформы Boosty.to, E2E-тесты без подготовки будут нестабильными и потребуют реальной авторизации. Для решения этой проблемы предлагается использовать **Mocking API**.

```
┌────────────────────────────────────────┐
│           Playwright (E2E Test)        │
└───────────────────┬────────────────────┘
                    │ Запускает Chromium с расширением
                    ▼
┌────────────────────────────────────────┐
│           Chromium (Страница Boosty)   │
│  - content.js                          │
│  - page_script.js                      │
└───────────────────┬────────────────────┘
                    │ Перехват сетевых запросов
                    ▼
┌────────────────────────────────────────┐
│   Playwright Request Interception      │
│   (Имитирует API Boosty: /post,        │
│    /reaction, профиль блогера)         │
└────────────────────────────────────────┘
```

*   **Перехват запросов (Network Interception):** Playwright позволяет перехватывать все запросы к доменам `api.boosty.to` и `boosty.to`. Мы настроем мок-сервер прямо внутри тестов:
    *   Запросы на получение списка постов будут возвращать фиксированный JSON (сгенерированный на основе реального `tags_info.json`), содержащий посты с разными уровнями подписки, лайками и тегами.
    *   Запросы на установку/удаление реакций (`/reaction`) будут возвращать `200 OK`, позволяя тестировать отправку лайков из сайдбара без реальной авторизации.
*   **Изоляция локального хранилища:** Перед каждым тестом `chrome.storage.local` и `sessionStorage` будут очищаться, обеспечивая независимость тестов друг от друга.

## 3. Этапы реализации

### Этап 1: Подготовка инфраструктуры
1.  Создание `package.json` в корне проекта (если отсутствует).
2.  Установка зависимостей: `npm install -D playwright vitest @testing-library/dom jsdom`.
3.  Настройка конфигурационных файлов `playwright.config.js` и `vitest.config.js`.

### Этап 2: Рефакторинг кода под юнит-тесты (при необходимости)
Для написания чистых юнит-тестов логические функции в `content.js` должны быть доступны для импорта в тестовую среду.
*   *Вариант решения:* В самом конце `content.js` добавить условный экспорт для среды тестирования (Node.js/Vitest), например:
    ```javascript
    if (typeof module !== 'undefined' && module.exports) {
      module.exports = { parseTitle, getGroupedTitles, ensureUserData, ... };
    }
    ```

### Этап 3: Написание юнит-тестов (Vitest)
*   Тестирование парсера названий (выделение глав, томов, очистка от лишнего текста).
*   Тестирование логики распределения тайтлов по вкладкам подписок.
*   Тестирование сортировок (проверка правильности сортировки по прогрессу, новизне и т.д.).

### Этап 4: Написание E2E-тестов (Playwright)
*   Написание базовых тестов: открытие страницы, проверка рендеринга сайдбара.
*   Написание интерактивных тестов: клики по чекбоксам, раскрытие групп, переход в настройки.
*   Написание комплексных тестов на синхронизацию: симуляция нативного лайка на странице Boosty и проверка автоматического выбора чекбокса в расширении.

## 4. Пример структуры E2E теста (Playwright)

Примерный код теста для проверки автоотметки прогресса при клике на чекбокс:

```javascript
import { test, expect, chromium } from '@playwright/test';
import path from 'path';

test('Смена состояния чекбокса отправляет лайк на Boosty', async () => {
  const pathToExtension = path.resolve(__dirname, '../');
  
  // Запуск браузера с расширением
  const context = await chromium.launchPersistentContext('', {
    headless: false, // Расширения работают только в headful режиме
    args: [
      `--disable-extensions-except=${pathToExtension}`,
      `--load-extension=${pathToExtension}`,
    ],
  });

  const page = await context.newPage();
  
  // Настройка перехвата API
  await page.route('**/v1/blog/lightfoxmanga/post/*/reaction*', route => {
    expect(route.request().method()).toBe('POST'); // Ожидаем POST запрос на лайк
    route.fulfill({ status: 200, contentType: 'application/json', body: '{}' });
  });

  await page.goto('https://boosty.to/lightfoxmanga');

  // Открываем сайдбар
  await page.click('#lf-trigger-btn');
  
  // Кликаем по первому чекбоксу в списке
  const checkbox = page.locator('.lf-chapter-checkbox').first();
  await checkbox.check();

  // Проверяем, что чекбокс стал отмеченным и получил класс лайка
  await expect(checkbox).toBeChecked();
  await expect(checkbox).toHaveClass(/lf-liked-checkbox/);

  await context.close();
});
```
