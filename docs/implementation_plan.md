# План реализации: Выбор провайдера синхронизации (Яндекс.Диск / WebDAV)

Этот план описывает изменения для интеграции возможности выбора провайдера синхронизации в интерфейсе настроек (Яндекс.Диск с предустановленным адресом или произвольный WebDAV) с динамическим переключением инструкций и полей ввода.

## User Review Required

> [!NOTE]
> Добавляется поле `provider` (`'yandex' | 'webdav'`) в настройки WebDAV. По умолчанию при наличии старой конфигурации провайдер будет определяться автоматически (если `baseUrl` содержит `yandex.ru`, то `yandex`, иначе `webdav`).

## Proposed Changes

### 1. Ядро конфигурации и провайдеры

#### [MODIFY] [content.js](file:///home/akai/projects/boosty-bookmark/content.js)
*   **Конфигурация по умолчанию:**
    *   Добавить свойство `provider: 'yandex'` в дефолтный объект `webdavConfig`.
*   **Автоопределение при загрузке:**
    *   В функции `loadWebDavConfig()` при чтении из `chrome.storage.local` проверять: если сохранён `baseUrl` и он содержит `yandex.ru`, проставлять `provider = 'yandex'`. Если `baseUrl` пустой, но провайдер не задан, также ставить `'yandex'`.
*   **Инициализация соединения:**
    *   В функциях `prepareWebDavConnection()` and `isWebDavConfigured()` использовать `https://webdav.yandex.ru` в качестве адреса сервера, если выбран провайдер `yandex`.

### 2. Интерфейс настроек (UI/UX)

#### [MODIFY] [content.js](file:///home/akai/projects/boosty-bookmark/content.js)
*   **Кнопки выбора провайдера:**
    *   Добавить группу переключателей (кнопки Яндекс.Диск / Другой WebDAV) в блок облачной синхронизации.
*   **Динамическая форма:**
    *   Если выбран **Яндекс.Диск**:
        *   Скрывать поле ввода "Адрес WebDAV-сервера" (так как адрес фиксирован).
        *   Отображать описание кода доступа: *"Сгенерированный пароль приложения Яндекс.Диска."*
        *   Отображать инструкцию для создания пароля приложения в Яндекс ID.
    *   Если выбран **Другой WebDAV**:
        *   Показывать поле ввода "Адрес WebDAV-сервера".
        *   Отображать стандартное описание кода доступа для Nextcloud/Owncloud.
        *   Отображать инструкцию для Nextcloud.
*   **Слушатели событий:**
    *   Добавить обработчики кликов на переключатели провайдеров, которые обновляют `webdavConfig.provider` и вызывают перерисовку настроек через `renderSettingsContent()`.
